> 由本對話彙整當前現狀與下一步五個階段。每階段獨立、可在新 session 接續執行。
> 起草時的快照：[git status](#附錄a-起草時的-git-status)、[最後一份報告](2026-05-25_agora-runtime整合測試實測.md)。

## 1. 目標（一句話）

做一個**中央 hub**，讓不同宿主機、不同廠牌的 AI agent（OpenClaw / headless Claude / Codex / Cursor …）和人類，在共享房間裡用 `@mention` + 持久收件匣協作，**晚加入或離線重連都能補齊脈絡，不斷層**。Phase 1 MVP = GUI + 邀請成功第一個 agent + 人類在房裡跟它聊起來、訊息不丟。

## 2. 目前狀態（對照設計文檔 §10）

| Phase | 內容 | 進度 |
|---|---|---|
| **Phase 0** Hub 骨架 | SQLite + REST + WSS + presence sweep + 埠占用接管 | 完成 |
| **Phase 1** MVP | GUI（Electron）+ 邀請流程 + 配對碼 + 引導 prompt + 人類↔agent | 基本完成（TC-01~10 全 PASS） |
| **Phase 2** 多 agent + mention + 不斷層 | 收件匣降噪、cursor 不斷層、recovery | 機制全做完，**但「兩個真實 daemon agent 同房協作」尚未端對端走過** |
| **Phase 3** agent↔agent + relate | 出處邊（reply_to/relates 自動回寫）、線程 | 已在 agora-runtime 落地 + 測試通過 |
| **Phase 4** 跨宿主機 + 異質廠牌 | 雲端部署、publicUrl、多廠牌真實接入 | 全本機，未跨機，未接真實 OpenClaw |
| **Phase 5** 打磨 | 鑑權強化、搜尋、歸檔 | 未開始 |

**已驗證可用的 API**：
- runtime backend 合約：`backend.respond({ self, trigger, context, members, roomId }) => { text, to?, replyTo?, relates? } | null`
- httpAgent 約定：HTTP POST `{ sessionKey, input, trigger }` → `{ content }`；session key `agora:<廠牌>:<roomId>:<agentId>`
- hub REST：`/invite` / `/pair` / `/rooms/:id/messages` / `/inbox?agent=&since=` / `/context/:id` / `/cursor`

**反模式**：
- 不要重新發明 backend 合約欄位（context 已由 assembler 算好）
- 不要假設 IDE turn-based agent 可被外部訊息喚醒（§3.1）
- 不要用本地時間戳排序，一律 hub `seq`（§3.3）

### Phase 2 ─ 真實接入 OpenClaw（**最關鍵的下一步**）

把「mock 跑通的迴圈」換成真實 OpenClaw 在本機 daemon。

**步驟**：
1. 在 OpenClaw gateway 暴露 `POST /turn` 端點：收 `{ sessionKey, input, trigger }` → 把 `input` 灌進 `sessionKey` 對應的 OpenClaw session 跑一回合 → 回 `{ content }`。
2. OpenClaw gateway 設 `tools.sessions.visibility:"all"` + `gateway.tools.allow`（前身硬知識）。
3. 啟動 hub → GUI 建房 → 邀請「openclaw」→ 拿加入金鑰。
4. 另一終端：
   ```powershell
   cd connectors/agora-runtime
   $env:OPENCLAW_GATEWAY_URL = "http://127.0.0.1:<gateway埠>/turn"
   node index.js --rest http://127.0.0.1:8787 --code <碼> --name OpenClaw --kind openclaw --backend openclaw
   ```
5. GUI 用人類 `@OpenClaw` 發三則：閒聊 / 多步任務 / `reply_to` 引用背景訊息。
6. 殺掉 runtime → GUI 再 @ 它一則 → 不帶 `--code` 同 `--state` 重啟，驗證補處理且不重複。

**驗收清單**：
- [ ] OpenClaw 真的回覆內容（不是 mock 也不是 echo）
- [ ] 回覆有正確 `reply_to`（觸發訊息）+ `relates`（背景訊息）
- [ ] 殺掉 → 再 @ → resume 後補處理 + 不重複（cursor_seq 接上）
- [ ] gateway 掛掉時 runtime 顯示明確錯誤，**不退回 echo**
- [ ] 寫測試報告 `.claude/development/reports/YYYY-MM-DD_openclaw真實接入實測.md`

**需要使用者協助**：提供 OpenClaw gateway 的 `/turn` 端點 + URL。

**反模式**：
- 不要為了讓它「動」就改 backend 合約；OpenClaw 回的不是 `{content}` 就去看 httpAgent 拆包邏輯
- gateway 失敗時不要默默回 echo（runtime 已做防呆，不要拆掉）

---

### Phase 3 ─ 兩個 daemon agent 同房協作

**做什麼**：在 Phase 2 通過後，啟動第二支 agora-runtime 入同房。

1. 第二支 runtime 用另一張加入金鑰入同房（可為另一個 OpenClaw session、headless Claude、或 mock）。
2. 人類 `@OpenClaw 請和 @Helper 一起做 X`，看兩者 ping-pong。
3. 驗證收件匣降噪：A 給 B 的訊息不會被 C 處理。
4. 驗證 `⟦∎COLLAB_FIN∎⟧` 終止 token 能正確收尾、不打轉。

**驗收**：兩個 daemon agent 來回 ≥3 回合完成一個小任務；transcript 寫進 report。

**反模式**：A 不該收到 `@B` 還回覆（壞了就查 [hub/src/db.js](../../../hub/src/db.js) `readInbox`）。

---

### Phase 4 ─ 跨宿主機部署

**做什麼**：hub 搬上有公網的 server，本機 OpenClaw 連雲端 hub。

1. server 上 `git clone` + `cd hub && npm install && npm start`；改 `config.json` 的 `publicUrl` 為對外 `https://your-host` 與 `wss://your-host`，secret 換成強隨機字串（或 `HUB_SECRET` env）。
2. 反代（nginx/caddy）打 TLS，後端打到 127.0.0.1:8787。
3. 本機 GUI 改連雲端 hub，secret 手填（`publicUrl` 設了後 `/app-config` 不再自動帶 secret，§8 安全行為）。
4. 本機 runtime `--rest https://your-host --code <碼>` 入房，重跑 Phase 2 驗收清單。

**驗收**：
- [ ] 本機 OpenClaw daemon 連雲端 hub 跑通 Phase 2 流程
- [ ] 故意斷網 30 秒 → 重連 → 補處理離線期間 @ 它的訊息
- [ ] hub `/health` 從公網可達；secret 沒洩漏在前端 source

**需要使用者協助**：提供有公網的 server（或選 VPS/Render/Fly 等部署位置）；DNS 與 TLS 憑證安排。

**反模式**：不要忘記改 secret 就上公網；不要用 `ws://` 在公網跑（必須 wss/TLS）。

---

### Phase 5 ─ 最終驗證 + 文檔收斂

1. Grep 確認沒人還在用本地時間戳排序（應只在日誌/presence 用 `Date.now()`，不應在訊息排序）。
2. Grep `echo-agent` 確認沒被新代碼依賴。
3. 跑完整測試：
   ```
   node .claude/development/tests/agora-runtime_integration_test.mjs --rest http://127.0.0.1:8787
   node .claude/development/tests/agora-runtime_manual_tc09_tc10.mjs --rest http://127.0.0.1:8787
   cd hub && npm run smoke
   ```
4. 匯總報告 `.claude/development/reports/YYYY-MM-DD_phase2-4完成總結.md`。

**驗收**：三套測試全 PASS；總結報告寫完。
